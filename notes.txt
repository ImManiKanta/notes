NACL vs Security groups in AWS
SG
-> Attached to instance level.
-> Source is ip address or security groups
-> U cant give deny rule
-> SG is stateful, it will automatically give outbound response to request

NACL
-> attached to subnet level
-> Source will be cidr of subnet
-> U can give deny rules
-> NACL is first layer of defence then SG
-> NACLs are stateless, you have to give outbound rule then only it will give response back to request
